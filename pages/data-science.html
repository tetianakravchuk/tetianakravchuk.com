ML Model Validation Case Study: Residential Property Values

This project is not presented as a production-ready appraisal model. It is a model validation case study showing how I evaluate model performance, compare algorithms, identify limitations, and reason about production risk, fairness, and data quality.

Validation resultRandom Forest — RMSE ~$290,919, MAE ~$184,569, R² ~0.517.

What the Model Results Actually Mean

The Random Forest model performed best among the tested models, but the result also shows why validation matters. An R² of approximately 0.517 means the model explains some meaningful signal, but it is not strong enough for high-stakes appraisal decisions without additional feature engineering, market-specific validation, fairness review, and monitoring.

ML QA Review: What I Would Test Before Production

  • Data quality: missing values, outliers, duplicated properties, and stale records.
  • Feature risk: location fields may proxy socioeconomic patterns.
  • Model performance: compare RMSE, MAE, R², and residual behavior by region and price band.
  • Fairness review: check whether errors are higher in specific neighborhoods or property groups.
  • Monitoring: track drift in market prices, interest rates, and regional housing patterns.
  • Release readiness: require baseline comparison, validation split documentation, and model-card style reporting.
